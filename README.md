🛒 Store Discord Bot

بوت ديسكورد احترافي ومبتكر مبني باستخدام Node.js ومكتبة Discord.js v14 مخصص لإدارة وتلقي المنتجات من موقعك الإلكتروني ونشرها تلقائياً داخل قنوات متجر ديسكورد بتصميمات جذابة وأزرار شراء تفاعلية مع دعم تتبع الروابط التسويقية (Affiliate).

🚀 الميزات الرئيسية (Key Features)

إعداد تلقائي ذكي (/setup-store):

أمر مخصص للمشرفين فقط لإنشاء قناة المتجر فوراً.

يعرض نافذة منبثقة تفاعلية (Modal) لإدخال وتخصيص اسم القناة.

يقوم بضبط صلاحيات القناة تلقائياً (منع الأعضاء من الكتابة أو التفاعل برموز تعبيرية خارجية، وتفعيل ميزة المشاهدة فقط).

عرض منتجات احترافي وأزرار شراء:

عرض المنتجات عبر بطاقات غنية مميزة (Discord Embeds) تشمل العنوان، الوصف، السعر، وصورة المنتج.

إضافة زر شراء مباشر (Buy Now) مرتبط بمنتجك.

تفاعل تلقائي للبوت برموز تعبيرية (Reactions) مثل 👍 و 🔥 لتسهيل تفاعل الأعضاء دون الحاجة لصلاحيات كتابة.

تكامل متقدم مع مواقع الويب (Webhook System):

يتضمن خادم ويب داخلي مدمج باستخدام Express.js يستمع للطلبات على المسار /api/webhook/new-product.

يمكن لموقعك الإلكتروني (أو ووكومرس، شوبيفاي، إلخ) إرسال المنتجات الجديدة للبوت تلقائياً لينشرها في ديسكورد فوراً.

دعم التسويق بالعمولة (Affiliate Support):

إمكانية إضافة معرف التسويق بالعمولة تلقائياً إلى روابط الشراء للتتبع وجمع الأرباح.

📋 متطلبات التشغيل (Prerequisites)

إصدار Node.js v16.11.0 أو أحدث.

حساب مطور ديسكورد وإنشاء تطبيق (Bot Application) على بوابة المطورين.

حساب على منصة Railway أو أي بيئة استضافة أخرى للتشغيل المستمر.

🛠️ التثبيت والتشغيل المحلي (Local Installation)

تحميل المشروع:
قم بتحميل الملفات إلى مجلد مشروعك.

تثبيت الحزم البرمجية:
افتح واجهة الأوامر (Terminal) في مجلد المشروع وقم بتشغيل:

npm install


إعداد ملف البيئة:
قم بإنشاء ملف باسم .env في المجلد الرئيسي وقم بتعبئة البيانات التالية:

DISCORD_TOKEN=your_bot_token_here
CLIENT_ID=your_application_client_id_here
PORT=3000
AFFILIATE_ID=your_affiliate_id
WEBHOOK_SECRET=your_secure_secret_token


بدء تشغيل البوت:

npm start


⚙️ شرح متغيرات البيئة (Environment Variables)

المتغير

الشرح

DISCORD_TOKEN

التوكن الخاص بالبوت الخاص بك من Discord Developer Portal.

CLIENT_ID

معرف البوت (Application ID).

PORT

المنفذ الذي سيعمل عليه خادم Express (الافتراضي هو 3000، وفي Railway يتم تحديده تلقائياً).

AFFILIATE_ID

المعرف التسويقي الذي يضاف لنهاية روابط الشراء لتتبع العمولات (مثال: ref=default_aff).

WEBHOOK_SECRET

رمز أمان مخصص لحماية رابط الويب هوك الخاص بك لتجنب إرسال طلبات عشوائية من الآخرين.

🌐 تكامل موقعك الإلكتروني (Webhook API)

لإرسال المنتجات تلقائياً من موقعك إلى سيرفر الديسكورد، قم بإرسال طلب من نوع POST إلى خادم البوت على المسار التالي:

رابط الويب هوك:

POST https://your-railway-app-url.up.railway.app/api/webhook/new-product


رأس الطلب (Headers):

Authorization: <قيمة الـ WEBHOOK_SECRET المحددة في ملف الـ .env>
Content-Type: application/json


جسم الطلب (Request Body - JSON):

{
  "guildId": "معرف_سيرفر_ديسكورد",
  "title": "ساعة ذكية فاخرة",
  "description": "ساعة ذكية مقاومة للماء مع تتبع دقات القلب وشاشة أموليد.",
  "price": "$59.99",
  "url": "https://yourstore.com/products/smart-watch",
  "image": "https://yourstore.com/images/smart-watch.jpg"
}


🚀 الرفع على منصة Railway

قم بإنشاء مستودع (Repository) جديد على GitHub وارفع إليه ملفات المشروع (index.js, package.json, .gitignore, README.md).

افتح موقع Railway.app وقم بتسجيل الدخول باستخدام حساب GitHub الخاص بك.

اضغط على New Project ثم اختر Deploy from GitHub repo.

اختر المستودع الخاص بمشروع البوت.

بمجرد إنشاء المشروع، انتقل إلى علامة التبويب Variables في إعدادات التطبيق على Railway وأضف متغيرات البيئة التالية:

DISCORD_TOKEN

CLIENT_ID

AFFILIATE_ID

WEBHOOK_SECRET
(ملاحظة: Railway سيتكفل بإضافة متغير PORT تلقائياً لتشغيل خادم الويب).

سيقوم Railway ببناء المشروع وتثبيت الحزم وتشغيل البوت تلقائياً ومستمر على مدار الساعة!
