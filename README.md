🛒 بوت متجر ديسكورد الاحترافي | Store Discord Bot

بوت ديسكورد ذكي واحترافي مبني باستخدام Node.js ومكتبة Discord.js v14 لتهيئة وإدارة متجرك الإلكتروني داخل سيرفر الديسكورد تلقائياً، مع دعم متكامل لاستقبال المنتجات عبر الويب هوك (Webhook) والتسويق بالعمولة (Affiliate).

🚀 الميزات الرئيسية (Key Features)

إعداد تلقائي ذكي (/setup-store):

أمر مخصص للمشرفين (Administrators) لإنشاء قناة المتجر وضبط صلاحياتها فوراً.

يعرض نافذة منبثقة تفاعلية (Modal) تطلب من المشرف تخصيص اسم القناة.

يقوم بتعديل صلاحيات القناة تلقائياً (منع الأعضاء من الكتابة أو إضافة تفاعلات خارجية، مع السماح بمشاهدة القناة وقراءة الأرشيف).

عرض منتجات احترافي مع أزرار تفاعلية:

تُعرض المنتجات على هيئة بطاقات غنية ومصممة بعناية (Discord Embeds) تشمل العنوان، الوصف، السعر، وصورة المنتج.

إضافة زر شراء مباشر (Buy Now 🛒) ينتقل بالمستخدم مباشرة إلى صفحة المنتج.

تفعيل تفاعلات تلقائية (Reactions) مثل 👍 و 🔥 لتسهيل تفاعل الأعضاء بشكل آمن.

تكامل متقدم مع مواقع الويب (Webhook API):

يحتوي البوت على خادم ويب داخلي مدمج باستخدام Express.js يستمع للطلبات على المسار /api/webhook/new-product.

يتيح لموقعك الإلكتروني (سواء كان ووكومرس، شوبيفاي، أو برمجة خاصة) إرسال المنتجات الجديدة تلقائياً ليتم نشرها فوراً في الديسكورد.

تتبع التسويق بالعمولة (Affiliate Support):

يقوم البوت تلقائياً بإلحاق رمز التسويق بالعمولة (ref=your_affiliate_id) بروابط الشراء لتتبع المبيعات والأرباح.

📋 متطلبات التشغيل (Prerequisites)

تثبيت بيئة Node.js (إصدار 16.11.0 أو أحدث).

حساب مطور ديسكورد وإنشاء تطبيق (Bot Application) من خلال بوابة المطورين.

حساب على منصة Railway (أو أي استضافة أخرى) للتشغيل المستمر.

🛠️ التثبيت والتشغيل المحلي (Local Installation)

تحميل ملفات المشروع:
قم بوضع الملفات التالية في مجلد مشروعك:

index.js

package.json

.gitignore

.env

تثبيت الحزم البرمجية:
افتح واجهة الأوامر (Terminal) في مجلد المشروع ونفّذ الأمر التالي:

npm install


إعداد ملف المتغيرات البيئية (.env):
قم بملء البيانات الخاصة بك داخل ملف .env:

DISCORD_TOKEN=your_bot_token_here
CLIENT_ID=your_application_client_id_here
PORT=3000
AFFILIATE_ID=your_affiliate_code
WEBHOOK_SECRET=your_secure_random_token


تشغيل البوت محلياً:

npm start


⚙️ شرح متغيرات البيئة (Environment Variables)

المتغير

الشرح

DISCORD_TOKEN

التوكن الخاص بالبوت من صفحة مطوري ديسكورد.

CLIENT_ID

معرف البوت الخاص بك (Application ID).

PORT

المنفذ الذي يعمل عليه خادم Express (الافتراضي هو 3000).

AFFILIATE_ID

المعرف التسويقي الذي يضاف تلقائياً إلى روابط المنتجات.

WEBHOOK_SECRET

رمز أمان مخصص لحماية رابط الويب هوك الخاص بك لتجنب الطلبات الخارجية غير المصرح بها.

🌐 ربط موقعك الإلكتروني بالبوت (Webhook API)

لإرسال المنتجات تلقائياً من موقعك إلى الديسكورد، قم بإرسال طلب من نوع POST إلى خادم البوت على الرابط التالي:

رابط الطلب (Endpoint):

POST https://your-railway-app-url.up.railway.app/api/webhook/new-product


رأس الطلب (Headers):

Authorization: <قيمة الـ WEBHOOK_SECRET المحددة في ملف .env>
Content-Type: application/json


جسم الطلب (Request Body - JSON):

{
  "guildId": "معرف_سيرفر_ديسكورد_الخاص_بك",
  "title": "سماعة رأس لاسلكية احترافية",
  "description": "سماعة محيطية ممتازة للألعاب وعازلة للصوت الخارجي مع بطارية تدوم 40 ساعة.",
  "price": "$89.99",
  "url": "https://yourstore.com/products/gaming-headset",
  "image": "https://yourstore.com/images/gaming-headset.jpg"
}


🚀 الرفع والتشغيل على منصة Railway

ارفع ملفات المشروع (index.js, package.json, .gitignore, README.md) إلى مستودع (Repository) جديد على GitHub.

سجل الدخول إلى موقع Railway.app باستخدام حساب GitHub الخاص بك.

اضغط على New Project ثم اختر Deploy from GitHub repo وحدد مستودع البوت الخاص بك.

بمجرد بدء إعداد المشروع، توجه إلى تبويب Variables في لوحة التحكم على Railway وأضف المتغيرات التالية يدويًا:

DISCORD_TOKEN

CLIENT_ID

AFFILIATE_ID

WEBHOOK_SECRET
(ملاحظة: سيتكفل نظام Railway بضبط متغير PORT بشكل تلقائي لربطه بخدمة الويب هوك).

سيقوم Railway ببناء مشروعك وتثبيت الحزم وبدء التشغيل التلقائي على مدار الساعة بنجاح!
